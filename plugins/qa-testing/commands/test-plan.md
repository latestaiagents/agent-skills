# /test-plan

Create a comprehensive test plan for your project or release.

## What I Need

Tell me:
- Project/feature being tested
- Timeline and milestones
- Team size and roles
- Risk areas
- Environment details

## What I'll Create

A complete test plan including:

1. **Scope** - What's in/out of testing
2. **Approach** - Test levels and types
3. **Schedule** - Timeline with milestones
4. **Resources** - People, environments, tools
5. **Risks** - Potential issues and mitigation
6. **Criteria** - Entry/exit conditions
7. **Deliverables** - Reports and artifacts

## Template Sections

```markdown
1. Introduction & Objectives
2. Scope (In/Out)
3. Test Strategy
4. Test Environment
5. Test Data Requirements
6. Schedule & Milestones
7. Resource Allocation
8. Risk Assessment
9. Entry/Exit Criteria
10. Deliverables
```

## Example

```
You: /test-plan

Project: E-commerce checkout redesign
Timeline: 4 weeks
Team: 2 QA engineers
Risks: Payment integration, mobile compatibility

Claude: [Generates complete test plan with schedule,
resource allocation, risk matrix, environment setup,
test types coverage, and success criteria]
```

## Output

Full document ready for:
- Stakeholder review
- Team alignment
- Project tracking
- Audit compliance
